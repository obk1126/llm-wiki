# CLAUDE.md — Wiki/ (정리된 지식)

LLM 컴파일러(Claude Code)가 Raw 소스를 소화해 만든 **재활용 가능한 지식**이 쌓이는 곳.
지식은 개념과 개체로 분해되어 서로 `[[링크]]`로 연결되고, 새 소스가 들어올 때마다 업데이트된다(복리 효과).

## 하위 폴더
- `Index/` — 전체 목차·인덱스. 개념 지도(MOC, Map of Content)를 둔다.
- `Log/` — 인제스트 로그(언제 무엇을 소화했는지) 시간순 기록.
- `Concepts/` — **개념** 노트 (예: PKM, RAG, COD 프로세스, 노트의 복리 효과).
- `Entities/` — **개체** 노트 (인물/도구/제품/조직, 예: 안드레 카파시, Graphify, Obsidian).

## 노트 규칙 (스키마)
- 파일명: 개념/개체 이름 그대로 (예: `LLM Wiki.md`). 위키링크 충돌 방지를 위해 고유하게.
- 필수 frontmatter:
  ```yaml
  ---
  type: concept | entity
  aliases: []          # 동의어/약칭
  sources: []          # 근거가 된 Raw 노트 [[링크]]
  related: []          # 연결된 다른 Wiki 노트 [[링크]]
  created: <날짜>
  updated: <날짜>
  tags: []
  ---
  ```
- 본문 구조: `## 정의` → `## 핵심 포인트` → `## 연결(왜 중요한가 / 나의 맥락과의 관계)` → `## 출처`.
- **나의 관점**: 단순 요약이 아니라 `_Core-Context.md`의 목적과 어떻게 연결되는지 한 줄 이상 적는다.
- 기존 노트가 있으면 새로 만들지 말고 **업데이트**하고 `updated`와 `sources`를 갱신한다.

## 업데이트 루프
인제스트마다: ① 개념/개체 추출 → ② 기존 노트와 병합/연결 → ③ `Index/` 갱신 → ④ `Log/`에 기록.
