# 운영 프로토콜 — ingest / query / lint / 인터뷰

## 셋업 인터뷰 가이드 (setup 모드)
다음 3문항을 **하나씩** 묻고, 답이 얕으면 한 번 더 파고든다(STT처럼 자유롭게 말하게 유도).

1. **나는 누구인가?** — 역할, 하는 일, 핵심 가치관, 강점.
   - 후속: "어떤 정체성으로 불리고 싶은가? 무엇을 가장 잘하나?"
2. **왜 기록하고 싶은가?** — 가장 잘 안 되는 것(병목), 궁극적 비전.
   - 후속: "기록이 없어서 가장 아쉬웠던 순간은? 1년 뒤 이 위키로 뭘 이루고 싶나?"
3. **어떤 아웃풋을 만들고 싶은가?** — 산출물 형태, 주요 오디언스, 채널.
   - 후속: "누구의 마음을 움직이고 싶나? 어디에 발행하나?"

답변 → `_Core-Context.md`의 해당 섹션과 루트 `CLAUDE.md`의 `{{WHO_AM_I}}/{{WHY_RECORD}}/{{OUTPUT_GOAL}}/{{AUDIENCE}}` 치환.

## ingest 프로토콜
1. 대상 결정: `Raw/`에서 `status: inbox`인 노트, 또는 사용자가 준 경로/URL.
2. 소스를 읽고 무엇인지 파악(시리즈면 묶어서 소스 페이지로 제안).
3. **목적 인터뷰 (Gold In)**: "이걸 왜 수집했나? 어떤 관점/연결을 넣을까?" — 답을 `purpose`에 반영.
4. 개념(Concept)/개체(Entity) 추출 → `Wiki/Concepts/`·`Wiki/Entities/`에 노트 생성/업데이트.
   - 기존 노트 있으면 병합·연결(`related`, `sources` 갱신). 새로 만들지 않는다.
5. `[[위키링크]]`로 상호 연결, `Wiki/Index/` MOC 갱신, `Wiki/Log/`에 인제스트 기록 추가.
6. Raw 노트 `status: ingested`로 변경 + 생성된 Wiki 노트 링크.
7. 시사점이 나오면 "Synthesis 저장할까요?" → 동의 시 `Synthesis/`에 저장.
- 주의: AI가 만든 위키 개념은 "재료"이며 사용자의 진짜 글과 다르다는 점을 명시.

## query 프로토콜
1. 질문 관련 `Wiki/`·`Synthesis/` 노트를 키워드 + 링크 그래프로 검색.
2. `graphify_out/graph.json` 있으면 노드/엣지 근거 활용("그래프 탐색 결과 기반").
3. 근거 노트를 `[[링크]]`로 인용하며 답변. 위키에 없으면 "근거 없음" 명시 + 인제스트 제안.

## lint 체크리스트
- [ ] 깨진 `[[링크]]` 탐지/수정
- [ ] 고아 노트(아무도 링크 안 함) → Index에 연결하거나 보고
- [ ] 중복 개념 병합(aliases 통합)
- [ ] frontmatter 누락 필드 보정
- [ ] `Wiki/Index/` 목차·MOC 재생성
- [ ] 오래된 `updated` 날짜와 실제 내용 불일치 점검
→ 변경사항을 사용자에게 요약 보고 후 적용.

## graphify 프로토콜
1. `pip3 show graphify` → 미설치면 사용자 확인 후 `pip3 install graphify` (정체 미검증 고지).
2. Wiki 폴더 대상 그래프 생성 → `graphify_out/`에 graph.json/html/report 저장.
3. 새 자료 추가 후엔 변경분만 재추출해 그래프 업데이트.
